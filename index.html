Recommended first run: stable model, 8K context, one GPU

Commands follow the platform switch in the top bar. Change it if this phone is only your setup screen.

8GB: LiquidAI LFM2.5-8B-A1B

The easiest 8GB start is a small official GGUF. It leaves room for the runtime, KV cache, and mistakes.

Quant: Q4_K_M Start context: 4096 Beginner-stable
Windows PowerShell 
.\llama-server.exe `
  -m C:\models\LFM2.5-8B-A1B-Q4_K_M.gguf `
  --ctx-size 4096 `
  -ngl auto `
  --flash-attn auto `
  --cache-type-k q8_0 `
  --cache-type-v q8_0 `
  --jinja `
  --port 8080
1. Install llama.cpp. On Windows, start with winget install llama.cpp. Manual install works too, but beginners often download the wrong zip.
2. Put the model in C:\models. Short paths avoid quoting problems. Rename the file to something simple after download.
3. Open http://localhost:8080. If the chat opens and answers once, the hard part is done.

The clean first setup.

Windows-first, because that is where the most beginner friction lives. Mac notes are one switch away.

Checked against llama.cpp b9637, June 2026

Install the runner

Run winget install llama.cpp. If you download manually, NVIDIA users want the Windows CUDA zip plus the matching CUDA runtime DLL zip from the same release.

Download one GGUF

Pick one model from the selector. Start with Q4 or Q3 on smaller GPUs. Do not start by downloading every quant.

Run the server

Use the generated command. Then open http://localhost:8080. The OpenAI-style endpoint is http://localhost:8080/v1.

Sanity checks before anything advanced

  • Run from PowerShell, not by double-clicking the exe.
  • Use .\llama-server.exe --list-devices to verify your GPU is visible.
  • Keep context at 4096 or 8192 for the first run.
  • Make one request in the built-in web UI before connecting Cursor, Continue, aider, or another client.
  • If you use an OpenAI-compatible client, set base URL to http://localhost:8080/v1 and use any non-empty API key.

Do not touch these yet

  • --ctx-size 180000: huge context eats VRAM fast.
  • --spec-type draft-mtp: only for MTP models after baseline works.
  • --tensor-split: multi-GPU tuning is not a first-run task.
  • --mmproj: vision requires a matching projector file.

Fix the symptom, not the flag.

Most failures are not mysterious. They are bad assets, invisible CPU fallback, too much context, or a model/runtime mismatch.

Beginner rescue map

The app flashes and closes

Run it from PowerShell. Use a short folder like C:\llama. If manual install, keep the matching CUDA DLL zip contents beside the exe.

GPU is idle

You probably installed a CPU build or the GPU is not visible. Run --list-devices, then use a CUDA/Vulkan/HIP build and -ngl auto.

Out of memory

Lower context before changing everything else. Try --ctx-size 4096, smaller quant, or --cache-type-k q8_0 --cache-type-v q8_0.

Generation is painfully slow

Check the log for GPU layer offload. If layers are on CPU, speed will crater. Reduce model size and context until it fully fits.

Output is weird or repetitive

Use an instruct/chat GGUF, keep --jinja, and test in the built-in web UI. Do not hand-write templates first.

Cursor or another client will not connect

Use http://localhost:8080/v1, not just port 8080. Check http://localhost:8080/health. Any non-empty API key is usually fine unless you configured one.

MTP does nothing or errors

Use MTP only with an MTP model. Current llama.cpp uses --spec-type draft-mtp, not old shorthand flags.

Vision fails

The projector must match the text model. Keep model.gguf and mmproj.gguf from the same repo/version.

Two GPUs are confusing

Start with one GPU. Multi-GPU split can help fit models, but it does not magically double speed. Tune after the single-GPU path works.

When to graduate.

Advanced flags are useful after you have a working baseline and a reason to add complexity.

Add one complexity at a time
Try MTP for speed

Only after your non-MTP command works. Use an MTP model and add --spec-type draft-mtp --spec-draft-n-max 3. Compare actual tokens/sec and acceptance rate.

Increase context

Move from 8K to 16K or 32K first. Huge 128K+ contexts are memory decisions, not defaults. KV cache grows with context and parallel slots.

Add vision

Download the matching mmproj for that exact vision model. If you do not need screenshots or image input, skip this.

Use two GPUs

Run --list-devices, then start with default layer split. Use --tensor-split only when you know the VRAM ratio and can test performance.

Plain-English glossary.

Enough vocabulary to avoid bad downloads and bad flags.

GGUF

The file format llama.cpp loads. If the model is not a .gguf, it is not the simple llama.cpp path.

Quantization

A smaller version of the model. Q4 is smaller and faster; Q8 is larger and usually better quality.

VRAM

Memory on your GPU. Model weights, context cache, and vision files all compete for it.

Context

How much text the model can keep in mind. More context costs more memory.

KV cache

The memory used to remember attention state while generating. It grows with context and concurrent requests.

MTP

Multi-token prediction. It can speed generation, but only with models that include MTP heads.

mmproj

The projector file needed for many vision models. It must match the model.

OpenAI-compatible endpoint

A local /v1 API that many tools can use instead of a cloud API.

GPU layers

How much of the model runs on the GPU. More layers on GPU usually means faster output.
